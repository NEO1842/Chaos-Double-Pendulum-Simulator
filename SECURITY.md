# 🔐 Security Policy

## 🧠 Supported Versions

This project is actively developed, and only the latest major version is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |

👉 Note: Older versions are not guaranteed to be stable due to frequent physics and simulation updates.

---

## 🚨 Reporting a Vulnerability

If you discover a security issue or unexpected behavior related to:

- Simulation instability
- Infinite loops or performance crashes
- Memory leaks during real-time rendering
- Unsafe or exploitable input handling

please report it responsibly.

---

## 📩 How to report

Please report vulnerabilities by:

- Opening a **private GitHub Security Advisory (preferred)**  
- Or creating an issue with the label: `security`

If the issue is sensitive, avoid public disclosure until it is fixed.

---

## ⏱ Response time

You can expect:

- Initial response: within a few days
- Confirmation: if valid or not
- Fix timeline: depends on severity

---

## ✅ What happens after reporting

If the vulnerability is accepted:

- It will be patched in a future update
- You may be credited (if you want)
- A fix release will be published

If declined:

- You will receive a reason why it is not considered a security issue

---

## ⚠️ Important note

Because this project simulates chaotic physical systems:

- Small parameter changes can cause large effects
- Some “unexpected behavior” may be part of the system’s nature, not a bug

Please include clear reproduction steps when reporting.
